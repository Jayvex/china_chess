# 🀄 中国象棋 (Chinese Chess)

基于 Web 的中国象棋游戏，支持**本地双人对战（PvP）**和 **人机对战（PvE）**两种模式。

## ✨ 功能特性

### 🎮 双人对战模式 (PvP)
- 两位玩家在同一设备上轮流走棋
- 红方先手，交替行棋
- 支持悔棋操作
- 完整的走棋记录与失子显示

### 🤖 AI 对战模式 (PvE)
- 内置 AI 引擎，采用**高级博弈搜索算法**
- **三档难度可选**：
  - 🟢 **简单** — 搜索深度 3，适合初学者
  - 🟡 **普通** — 搜索深度 5（默认），适合有一定基础的玩家
  - 🔴 **困难** — 搜索深度 7，具有较强棋力
- 内置**开局库**，包含 80+ 种专业开局变化
- 可选择**执红先手**或**执黑后手**
- AI 思考时显示动态提示
- 支持**悔两步**（撤销玩家+AI 各一步）

### 🎯 完整象棋规则
- 7 种棋子走法：帅/将、仕/士、相/象、馬、車、炮、兵/卒
- 九宫格限制（将/帅、仕/士）
- 蹩马腿、塞象眼
- 炮翻山吃子
- 兵/卒过河后可横移
- 将军检测、将死判定、和棋判定
- 对面将规则（将/帅不能照面）

### 🎨 视觉设计
- SVG 木纹棋盘渲染，新中式暗色典雅风格
- 金色主题配色
- 棋子选中高亮（金色边框）
- 合法走法提示：绿点（移动）+ 红虚线圈（吃子）
- 回合指示器带呼吸灯动画
- 响应式布局，适配不同屏幕尺寸

## 🚀 快速开始

### 环境要求
- Node.js >= 18
- npm / pnpm / yarn

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/Jayvex/china_chess.git
cd china_chess

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器访问 `http://localhost:5173` 即可开始游戏。

### 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist/` 目录，可部署至任意静态托管服务。

## 📖 使用指南

### 选择游戏模式

页面顶部提供两个模式按钮：

| 按钮 | 说明 |
|------|------|
| 👥 **双人对战** | 本地两人轮流走棋 |
| 🤖 **AI 对战** | 点击后弹出设置面板 |

### AI 对战设置

点击「AI 对战」后弹出设置面板：

```
┌─────────────────────┐
│ 难度                │
│   🟢 简单           │
│   🟡 普通  ← 当前   │
│   🔴 困难           │
│ ─────────────────── │
│ 执子                │
│   执红（先手）       │
│   执黑（后手）       │
└─────────────────────┘
```

1. 先选择难度（简单/普通/困难）
2. 再选择执子颜色
3. 点击即开始对局

### 操作方式

- **选棋**：点击己方棋子，显示金色高亮和合法走法提示
- **走棋**：点击绿色圆点（移动）或红色虚线圈（吃子）完成走棋
- **取消选择**：点击其他位置或再次点击已选棋子
- **悔棋**：右侧边栏「悔棋」按钮（PvE 模式下为「悔两步」）
- **新局**：右侧边栏「新局」按钮重新开始

### 胜负判定

- **将死对方将/帅** → 胜利弹窗
- **无子可动且未被将军** → 和棋（逼和）

## 🧠 AI 引擎说明

### 搜索算法

| 算法 | 说明 |
|------|------|
| **Alpha-Beta 剪枝** | 核心博弈树搜索，大幅减少需要评估的节点数 |
| **迭代加深** | 逐步增加搜索深度，确保在时间内返回最佳走法 |
| **PVS（主要变例搜索）** | 先用零窗口搜索，失败后重新全窗口搜索，提升效率 |
| **空着剪枝** | 跳过一手后若仍能产生剪枝，则直接剪掉该分支 |
| **静态搜索** | 深度耗尽后只搜索吃子走法，解决水平线效应 |

### 搜索优化

| 优化 | 说明 |
|------|------|
| **置换表** | 缓存已搜索局面的结果，避免重复计算（50 万条容量） |
| **杀手走法启发** | 记住每个深度导致剪枝的非吃子走法，优先搜索 |
| **历史启发表** | 记录走法的历史效果，高频有效的走法优先排序 |
| **MVV-LVA 排序** | 吃子走法按「最有价值受害者 - 最低价值攻击者」排序 |
| **走法排序** | 置换表 > 吃子 > 杀手 > 历史 > 将军，综合排序提升剪枝率 |

### 开局库

- 包含 **80+ 种**专业开局变化
- **红方开局**：中炮局（屏风马/反宫马/直车/横车/巡河车/过河车/盘头马）、飞相局、仙人指路、起马局、士角炮、过宫炮、仕角炮、巡河炮等
- **黑方应法**：屏风马、反宫马、单提马、顺手炮、列手炮、卒底炮、飞象应法等
- 走法经过**合法性验证**（符合蹩马腿、塞象眼等规则）
- 按权重随机选择，增加开局变化性

### 评估函数

| 评估维度 | 说明 |
|----------|------|
| **基础分值** | 車(1000) > 炮(500) > 馬(400) > 相/象(200) = 仕/士(200) > 兵/卒(100) |
| **位置价值表** | 每种棋子独立的 10×9 矩阵，精确评估棋子在不同位置的价值 |
| **机动性评估** | 评估双方可走步数之差，鼓励活跃走法 |
| **将/帅安全** | 被将军一方扣分，鼓励进攻和防守 |
| **配合评估** | 双车、车炮、马炮配合加分，鼓励子力协同 |

### 性能参考

| 难度 | 搜索深度 | 典型耗时 |
|------|----------|----------|
| 🟢 简单 | 3 | < 100ms |
| 🟡 普通 | 5 | ~200ms-1s |
| 🔴 困难 | 7 | ~1s-10s |
